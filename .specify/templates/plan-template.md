## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

验证以下 Constitution 原则的合规性：

- **Type-Safety First**: 所有新代码是否包含完整的 TypeScript 类型定义？是否避免了 `any` 类型？
- **Service Layer Architecture**: 业务逻辑是否在服务层实现？API 路由是否仅负责请求/响应处理？
- **AI Model Abstraction**: 如果涉及 AI 功能，是否通过 `lib/ai/model-router.ts` 访问？是否支持模型切换？
- **Platform Agnostic Design**: 如果涉及平台集成，是否遵循统一的接口契约？是否易于扩展新平台？
- **API-First Development**: 是否先定义和实现 API 端点，再构建前端组件？
- **Testing Discipline**: 关键业务逻辑是否包含测试计划？测试是否独立运行？
