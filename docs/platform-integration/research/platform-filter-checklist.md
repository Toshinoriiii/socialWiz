# 平台筛选检查清单

**用途**: 用于判断平台是否支持个人开发者，以及评估接入成本

## 检查项

### 1. 开发者类型支持

- [ ] **个人开发者支持**
  - 检查平台是否明确支持个人开发者注册
  - 查看注册页面是否有"个人开发者"选项
  - 确认个人开发者可以申请API访问权限

- [ ] **企业开发者要求**
  - 检查是否需要企业认证
  - 确认是否需要营业执照等企业资质
  - 评估企业认证的复杂度和成本

### 2. 认证要求

- [ ] **身份验证要求**
  - 是否需要实名认证
  - 是否需要提交身份证明
  - 认证审核时间

- [ ] **特殊资质要求**
  - 是否需要特殊行业资质
  - 是否需要特殊许可证
  - 是否需要其他认证材料

### 3. 费用评估

- [ ] **免费额度**
  - 是否有免费API调用额度
  - 免费额度的限制
  - 超出免费额度后的费用

- [ ] **付费方案**
  - 付费方案的价格
  - 付费方案的功能差异
  - 付费方案的必要性

### 4. 技术复杂度

- [ ] **API复杂度**
  - API文档的完整性
  - API的易用性
  - 集成难度评估

- [ ] **维护成本**
  - API的稳定性
  - 文档更新频率
  - 技术支持质量

### 5. 限制条件

- [ ] **API调用限制**
  - 调用频率限制
  - 调用量限制
  - 限制是否合理

- [ ] **内容限制**
  - 内容类型限制
  - 内容长度限制
  - 内容审核要求

## 筛选标准

### 推荐接入条件

✅ **满足以下条件可以推荐接入**:
- 明确支持个人开发者
- 不需要企业认证
- 有合理的免费额度或费用较低
- API文档完整，技术复杂度适中
- 限制条件合理，不影响正常使用

### 不推荐接入条件

❌ **满足以下条件不推荐接入**:
- 仅支持企业开发者
- 需要企业认证或特殊资质
- 费用过高或没有免费额度
- API文档不完整，技术复杂度高
- 限制条件过于严格，影响正常使用

## 使用说明

1. 访问平台官方文档网站
2. 按照检查清单逐项检查
3. 记录检查结果
4. 根据筛选标准判断是否推荐接入
5. 在调研报告中记录评估结果
