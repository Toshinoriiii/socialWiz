# 微博数据能力路线图

本文件对应 `specs/main` 中「数据（迭代）」范围；只读数据模块待产品定义后再实现。

## 目标

在用户已完成微博 OAuth 绑定的前提下，于本平台内只读拉取微博侧数据（如个人时间线、互动统计等），须遵守：

- 微博开放平台接口权限与 `scope`；
- API 频次限制；
- 用户数据合规与脱敏展示。

## 里程碑（建议）

1. **M1**：基于已存 `access_token` 调用 `users/show` 等基础接口，在账户页展示头像、粉丝数等（部分已由绑定流程覆盖）。
2. **M2**：按业务选型接入时间线类接口（需单独申请权限），服务端经 `lib/services/` 聚合后暴露 REST。
3. **M3**：缓存与定时同步策略，降低实时调用压力。
